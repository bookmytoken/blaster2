# ShieldGuard: System Requirements & GitHub Upload Guide

This document defines the software dependencies required for your professor (or any evaluator) to download and run the project locally, as well as instructions on what files must be uploaded to your GitHub repository.

---

## 1. System Requirements (For the Professor/Evaluator)
To successfully run the **ShieldGuard** environment, the host machine must have the following installed:

### Required Software
*   **Git**: To clone the repository.
*   **Node.js (v18.x or higher)**: Required to run the custom Express backend (`mock_trap.js`).
    *   *Download:* [nodejs.org](https://nodejs.org/)
*   **Flutter SDK (v3.22 or higher)**: Required to compile and run the frontend dashboard.
    *   *Download:* [flutter.dev](https://docs.flutter.dev/get-started/install)
*   **Google Chrome** or **Microsoft Edge**: Required to view the Flutter Web interface.

### Network Requirements
*   A stable internet connection is required the very first time to download Node modules and Flutter package dependencies via `npm install` and `flutter pub get`.
*   Ports **5001** and **8080** must be available (not blocked by local firewalls or other applications).

---

## 2. GitHub Upload Guide

When you push this project to GitHub, you do **NOT** need to upload every single file on your hard drive. Many folders contain heavy, automatically generated code (like the `node_modules` folder, which can be hundreds of megabytes).

I have already updated your `.gitignore` file to automatically block these heavy files.

### 🟢 DO Upload These Files (Required)
These are your core project files. If you use standard Git commands (`git add .`), Git will automatically select these:
*   `lib/` (The entire folder containing your Flutter Dart code)
*   `web/` (The folder containing your index.html and web assets)
*   `firebase/functions/` (The folder containing `mock_trap.js` and `package.json`)
*   `pubspec.yaml` and `pubspec.lock` (Flutter dependency list)
*   `README.md` and `REQUIREMENTS.md`
*   `proof/` and `assets/` (Your screenshots folder)
*   `.gitignore` (The file telling GitHub what to ignore)

### 🔴 DO NOT Upload These Folders (Ignored by `.gitignore`)
Do not forcefully try to upload these. The professor's machine will recreate them automatically when they run `npm install` and `flutter run`.
*   `firebase/functions/node_modules/` (Very heavy, automatically generated by Node)
*   `.dart_tool/` (Automatically generated by Flutter)
*   `build/` (Compiled output files)
*   `.idea/` or `.vscode/` (Your personal editor settings)

---

## 3. How to Push to GitHub (Terminal Commands)

If you haven't yet initialized the repository, run these exact commands in your terminal inside the `shieldguard` folder:

```bash
# 1. Initialize Git (only if you haven't already)
git init

# 2. Add all the required files (the .gitignore will automatically block the bad ones)
git add .

# 3. Save the submission
git commit -m "Final Hackathon Submission: ShieldGuard"

# 4. Link to your empty GitHub repository (Replace the URL with your actual GitHub link)
git branch -M main
git remote add origin https://github.com/YourUsername/YourRepoName.git

# 5. Push the code to GitHub!
git push -u origin main
```
